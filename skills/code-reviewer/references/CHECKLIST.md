# コードレビュー チェックリスト

## Python

### 基本
- [ ] PEP 8 スタイルガイドに準拠しているか
- [ ] 型ヒント (type hints) が適切に使われているか
- [ ] docstring が関数/クラスに記載されているか
- [ ] f-string を使用しているか（% や .format() ではなく）

### セキュリティ
- [ ] `eval()` / `exec()` を使用していないか
- [ ] SQL クエリにパラメータバインディングを使用しているか
- [ ] 機密情報が環境変数から取得されているか
- [ ] `pickle` の安全でないデシリアライゼーションがないか

### パフォーマンス
- [ ] リスト内包表記を適切に使用しているか
- [ ] ジェネレータを大量データ処理に使用しているか
- [ ] `asyncio` を I/O バウンド処理に活用しているか
- [ ] 不要なグローバル変数がないか

## JavaScript / TypeScript

### 基本
- [ ] `const` / `let` を使用しているか（`var` ではなく）
- [ ] Optional chaining (`?.`) を活用しているか
- [ ] `async/await` を使用しているか（`.then()` チェーンではなく）
- [ ] TypeScript の場合、`any` 型を避けているか

### セキュリティ
- [ ] `innerHTML` の代わりに `textContent` を使用しているか
- [ ] ユーザー入力をサニタイズしているか
- [ ] CORS 設定が適切か
- [ ] 依存パッケージのバージョンが最新か

### パフォーマンス
- [ ] 不要な再レンダリングを防いでいるか (React の場合)
- [ ] バンドルサイズに注意しているか
- [ ] メモリリークがないか（イベントリスナーの解除等）
- [ ] 適切な debounce / throttle を使用しているか

## 共通

### テスト
- [ ] ユニットテストがあるか
- [ ] エッジケースのテストがあるか
- [ ] モックが適切に使われているか
- [ ] テストが独立して実行可能か

### ドキュメント
- [ ] README が更新されているか
- [ ] 破壊的変更がある場合、マイグレーションガイドがあるか
- [ ] API ドキュメントが正確か
